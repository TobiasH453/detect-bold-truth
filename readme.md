# AIsore

AIsore is a mobile-first AI detection web app. It lets users analyze photos,
videos, text, and links, then returns a mock confidence score, verdict, and
suspected generator model.

The current detection flow is UI-focused and uses an in-browser mock data layer
stored in `localStorage`. Supabase is present for waitlist/admin support, but
the core analysis screens do not call a real detection backend yet.

## Stack

- React 19
- TanStack Start and TanStack Router
- Vite 7
- Tailwind CSS v4
- shadcn/ui and Radix UI primitives
- Supabase for waitlist/admin data
- Cloudflare/Vercel-compatible app configuration through the Vite setup

## Getting Started

Install dependencies:

```bash
npm install
```

Run the dev server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview a production build:

```bash
npm run preview
```

Bun can also be used in this repo because `bun.lockb` is present:

```bash
bun install
bun run dev
```

## Environment Variables

The main detect/history/result flow works without environment variables because
it uses `src/lib/profile.ts` mocks.

Supabase-backed waitlist/admin features need:

```bash
VITE_SUPABASE_URL=
VITE_SUPABASE_PUBLISHABLE_KEY=
SUPABASE_URL=
SUPABASE_PUBLISHABLE_KEY=
SUPABASE_SERVICE_ROLE_KEY=
ADMIN_PASSWORD=
```

`VITE_SUPABASE_*` is used by browser code. `SUPABASE_*` and
`ADMIN_PASSWORD` are used by server-side admin functions.

## Mock Detection Layer

`src/lib/profile.ts` is the current data and analysis boundary. It exports:

- `analyzeMock(input)` for creating a fake analysis result
- `listAnalyses()` for local history
- `getAnalysis(id)` for result lookup
- shared types such as `ContentType`, `Verdict`, and `Analysis`

To connect a real backend, keep the route/component contracts stable and replace
the mock functions behind this module.

## Supabase Waitlist

The waitlist table is defined in:

```text
supabase/migrations/20260513201724_1e0ebe2f-6761-44ef-8537-cbce979de743.sql
```

The admin route reads from `waitlist_signups` through
`src/lib/admin.functions.ts` using the service role key, so keep
`SUPABASE_SERVICE_ROLE_KEY` server-only.
