# AIsore — UI Handoff

This project is a **UI-only / design** deliverable. All backend wiring has been
removed and replaced with in-memory mocks (persisted to `localStorage` so the
flows feel real for design review). A fresh dev or AI agent can now plug in any
backend they like (Supabase, Firebase, custom Node/Postgres, etc.) by replacing
the mock implementations.

---

## Stack

- **Framework**: TanStack Start v1 (React 19, file-based routing under `src/routes/`)
- **Build**: Vite 7
- **Styling**: Tailwind CSS v4 via `src/styles.css` (semantic design tokens in `oklch`)
- **UI primitives**: shadcn/ui (`src/components/ui/*`)
- **Notifications**: `sonner`
- **Icons**: `lucide-react`

If you'd rather host this as a plain SPA, you can swap TanStack Start for plain
Vite + React Router with modest changes (route files convert 1:1).

## Where the seams are

Every backend touchpoint lives behind one of these three modules. **Replace
these and the rest of the app stays intact.**

### 1. `src/lib/auth-context.tsx` — Authentication

Mock that stores a fake user in `localStorage`. Any email + password "works".

**Contract a real implementation must satisfy:**
```ts
useAuth() => {
  user: { id: string; email: string } | null;
  loading: boolean;
  signIn(email: string, password: string): Promise<{ error: string | null }>;
  signUp(email: string, password: string): Promise<{ error: string | null }>;
  signOut(): Promise<void>;
}
```

### 2. `src/lib/profile.ts` — Profile, plan, credits, history

Mock data layer with localStorage persistence. Exports the canonical types
used everywhere in the UI.

**Types (keep these stable, or refactor consumers):**
```ts
type ContentType = "photo" | "video" | "text" | "link";
type Plan = "free" | "pro";
type Verdict = "likely_ai" | "likely_human" | "uncertain";

interface Profile {
  id: string;
  email: string;
  plan: Plan;
  credits_remaining: number;
  credits_reset_at: string; // ISO
}

interface Analysis {
  id: string;
  user_id: string;
  content_type: ContentType;
  input_preview: string;
  score: number;          // 0..100
  verdict: Verdict;
  suspected_model: string;
  credits_spent: number;
  created_at: string;     // ISO
}
```

**Functions to implement against your backend:**
```ts
fetchProfile(userId, email?) : Promise<Profile>
setPlan(userId, plan)        : Promise<Profile>      // also resets credits
listAnalyses(userId)         : Promise<Analysis[]>   // newest first
getAnalysis(id)              : Promise<Analysis | null>

// The "do the analysis" call. Should:
//   1. Validate input (text: 1..1000 words)
//   2. Atomically debit CREDIT_COST[contentType] from the user's profile
//   3. Run the actual AI-detection (currently mocked with a deterministic hash)
//   4. Persist + return the Analysis row
analyzeMock(input: {
  userId: string;
  contentType: ContentType;
  preview?: string;
  text?: string;
}): Promise<
  | { ok: true; analysis: Analysis }
  | { ok: false; error: string }
>;
```

**Credit costs** (defined in `CREDIT_COST`):
- photo: **4**
- video: **10**
- text: **2**
- link: **2**

**Plan rules:**
- Free → 30 credits / month
- Pro → 300 credits / month, $3.99/mo
- Credits reset 30 days after `credits_reset_at`

### 3. (Removed) `src/lib/detect.functions.ts`

Was a TanStack `createServerFn`. Deleted — its logic is now in `analyzeMock`.

## Design system

All colors / radii / shadows / fonts are tokenized in `src/styles.css` using
Tailwind v4's `@theme` + `oklch` variables. Components use semantic class names
(`bg-background`, `text-foreground`, `bg-primary`, `shadow-pop`, etc.) — no
raw hex values in component code.

Fonts: **Space Grotesk** (display) + **Inter** (body), loaded from Google Fonts
in `__root.tsx`.

Logo asset: `src/assets/aisore-logo.png` (used by `src/components/Logo.tsx`).

## Suggested next steps for whoever picks this up

1. Pick a backend (Supabase is closest to the original schema; anything works).
2. Implement the auth contract in `src/lib/auth-context.tsx`.
3. Implement the data + analysis contract in `src/lib/profile.ts`.
4. Wire a real AI-detection model behind `analyzeMock` (or rename it).
5. Delete the `localStorage` persistence helpers — they're only for the mock.
